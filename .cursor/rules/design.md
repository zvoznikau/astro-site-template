# Design Rules

**Связанные документы**: См. `.cursor/rules/language.md` для правил языка и локализации

## Общие принципы

### Кросс-платформенность

- **Сайт должен работать одинаково на всех системах**: macOS, Windows, Linux
- **Тестировать на разных браузерах**: Chrome, Firefox, Safari, Edge
- **Избегать системно-специфичных решений**: Не использовать функции доступные только на одной ОС
- **Кроссбраузерная совместимость**: Проверять на всех основных браузерах

### Адаптивный дизайн

- **Mobile-first подход**: Сначала мобильная версия, затем десктоп
- **Breakpoints**: Использовать стандартные breakpoints Tailwind:
  - `sm`: 640px
  - `md`: 768px
  - `lg`: 1024px
  - `xl`: 1280px
  - `2xl`: 1536px
- **Тестирование на реальных устройствах**: Проверять на разных размерах экранов
- **Touch-friendly**: Элементы должны быть удобны для touch-управления (минимум 44x44px)

## Использование компонентов и стилей

### Приоритет существующих компонентов

**Строгое правило**: Использовать существующие компоненты фреймворков.

1. **daisyUI компоненты в приоритете**:
   - Использовать готовые компоненты daisyUI (Button, Card, Navbar, etc)
   - Не переопределять стили daisyUI компонентов
   - Использовать daisyUI темы и токены

2. **Tailwind CSS утилиты**:
   - Использовать Tailwind utility classes
   - Избегать кастомных CSS когда можно использовать Tailwind
   - Использовать responsive utilities (`sm:`, `md:`, `lg:`)

3. **Не трогать код фреймворков**:
   - Не изменять файлы node_modules
   - Не переопределять стили фреймворков напрямую
   - Не модифицировать исходный код библиотек

### Кастомные стили

**Только когда необходимо**:

1. **Отдельные CSS файлы**:
   - Кастомные стили в отдельных файлах в `src/styles/`
   - Не смешивать кастомные стили с глобальными
   - Использовать CSS modules или scoped styles

2. **Структура стилей**:
   ```
   src/styles/
   ├── global.css          # Только Tailwind imports
   ├── custom.css          # Кастомные стили (если нужны)
   └── components/         # Стили для специфичных компонентов
       └── hero.css
   ```

3. **Когда создавать кастомные стили**:
   - Когда daisyUI/Tailwind не покрывают нужный функционал
   - Для специфичных анимаций
   - Для уникальных дизайн-элементов проекта

4. **Когда НЕ создавать кастомные стили**:
   - Если можно использовать daisyUI компонент
   - Если можно использовать Tailwind utility
   - Если можно комбинировать существующие классы

## Темы (Light/Dark)

### Поддержка тем

- **Обязательная поддержка**: Светлая и темная тема
- **daisyUI темы**: Использовать встроенную систему тем daisyUI
- **Переключение тем**: Через `data-theme` атрибут на `<html>`

### Реализация

```html
<!-- В layout -->
<html lang="en" data-theme="light">
  <!-- или -->
  <html lang="en" data-theme="dark">
</html>
```

### Правила для тем

1. **Использовать daisyUI токены**:
   - `bg-base-100` - фон
   - `bg-base-200` - вторичный фон
   - `text-base-content` - основной текст
   - `text-base-content/70` - вторичный текст
   - `btn-primary`, `btn-secondary` - кнопки

2. **Не хардкодить цвета**:
   ```html
   <!-- Плохо -->
   <div class="bg-white dark:bg-gray-900">
   
   <!-- Хорошо -->
   <div class="bg-base-100">
   ```

3. **Тестировать обе темы**:
   - Проверять читаемость в обеих темах
   - Убедиться что контраст достаточен
   - Проверять что все элементы видны

## Конфигурация и переменные

### Единый файл конфигурации

**Все переменные сайта в одном файле**: `src/config/brand.ts`

### Структура конфигурации

```typescript
// src/config/brand.ts
export const brand = {
  // Основная информация
  name: 'Site Name',
  description: 'Site description',
  
  // Контакты
  contact: {
    email: 'contact@example.com',
    phone: '+1 234 567 8900',
    address: '123 Street, City, Country',
  },
  
  // URL и ссылки
  urls: {
    website: 'https://example.com',
    github: 'https://github.com/example',
    twitter: 'https://twitter.com/example',
    linkedin: 'https://linkedin.com/company/example',
  },
  
  // CTA
  cta: {
    primary: 'Get Started',
    secondary: 'Learn More',
  },
  
  // Тема
  theme: {
    default: 'light',
    available: ['light', 'dark'],
  },
  
  // SEO
  seo: {
    title: 'Site Title',
    description: 'SEO description',
    keywords: ['keyword1', 'keyword2'],
    ogImage: '/og-image.jpg',
  },
}
```

### Использование конфигурации

```astro
---
import { brand } from '../config/brand'
---

<h1>{brand.name}</h1>
<p>{brand.description}</p>
<a href={brand.urls.website}>{brand.cta.primary}</a>
```

### Правила

1. **Все переменные в brand.ts**: Не хардкодить значения в компонентах
2. **Единый источник истины**: Все данные из конфигурации
3. **Типизация**: Использовать TypeScript для типов конфигурации
4. **Не дублировать**: Не копировать значения в разные места

## Кодстайл и дизайн-система

### Соблюдение кодстайла проекта

1. **Prettier форматирование**: Всегда запускать `pnpm format`
2. **ESLint правила**: Следовать правилам проекта
3. **TypeScript strict mode**: Использовать строгий режим
4. **Компонентный подход**: Разбивать на переиспользуемые компоненты

### Дизайн-система

1. **Консистентность**:
   - Использовать одинаковые отступы (spacing)
   - Единые размеры шрифтов
   - Консистентные цвета через daisyUI

2. **Типографика**:
   - Использовать Tailwind typography utilities
   - Следовать иерархии заголовков (h1, h2, h3)
   - Консистентные размеры текста

3. **Spacing**:
   - Использовать Tailwind spacing scale
   - Консистентные отступы между секциями
   - Единые padding/margin значения

4. **Компоненты**:
   - Переиспользуемые компоненты в `src/components/ui`
   - Секции страниц в `src/components/sections`
   - Избегать дублирования кода

## Адаптивность

### Mobile-first подход

1. **Начинать с мобильной версии**:
   - Сначала стили для мобильных устройств
   - Затем добавлять стили для больших экранов через breakpoints

2. **Responsive utilities**:
   ```html
   <!-- Mobile-first -->
   <div class="text-sm md:text-base lg:text-lg">
   <div class="grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3">
   ```

3. **Flexible layouts**:
   - Использовать `flex` и `grid` для адаптивных layouts
   - Использовать `clamp()` для fluid typography
   - Избегать фиксированных размеров

### Тестирование

1. **Размеры экранов**:
   - Mobile: 320px - 640px
   - Tablet: 640px - 1024px
   - Desktop: 1024px+
   - Large Desktop: 1280px+

2. **Инструменты**:
   - Chrome DevTools responsive mode
   - Реальные устройства
   - BrowserStack или аналогичные сервисы

## Язык и локализация

**Подробная документация**: См. `.cursor/rules/language.md`

### Основные правила

- **Язык по умолчанию**: Английский деловой стиль с акцентом на USA
- **Ориентация**: USA / UK / GE, но язык только английский
- **Русский язык запрещен**: Ни в коде, ни на сайте
- **Google Translate**: Обязательная интеграция для удобства пользователей
- **Проверка на кириллицу**: Обязательна перед коммитом

## Запрещенные практики

- **Хардкодить значения**: Использовать конфигурацию вместо хардкода
- **Модифицировать фреймворки**: Не трогать код библиотек
- **Inline styles**: Использовать Tailwind classes
- **Фиксированные размеры**: Использовать responsive utilities
- **Хардкодить цвета**: Использовать daisyUI токены
- **Дублировать код**: Создавать переиспользуемые компоненты
- **Игнорировать темы**: Всегда поддерживать light/dark
- **Системно-специфичные решения**: Использовать кросс-платформенные подходы
- **Использовать русский язык**: Строго запрещено
- **Кириллица в коде**: Строго запрещена
- **Игнорировать проверку на кириллицу**: Обязательна перед коммитом

## Best Practices

1. **Использовать существующие компоненты**: daisyUI в приоритете
2. **Кастомные стили в отдельных файлах**: Если нужны
3. **Все переменные в brand.ts**: Единый источник истины
4. **Mobile-first**: Начинать с мобильной версии
5. **Поддержка тем**: Light и dark обязательно
6. **Кросс-платформенность**: Работать везде одинаково
7. **Тестирование**: Проверять на разных устройствах и браузерах
8. **Консистентность**: Следовать дизайн-системе проекта
